## Фаза 4+ — Итерации по метрикам

План уточняется после первых оценок. Ориентиры:

- **precision / ndcg** → лучше персонализация, учёт фидбека, ранжирование
- **diversity** → MMR / жанровое разнообразие в выдаче
- **coverage** → не залипать на топ-популярном, exploration
- **latency** → Redis-кэш готовых recs per user, precompute в pipeline

### Идеи на будущее (если понадобится)
- Qdrant для item/user embeddings
- ALS / implicit feedback из interactions
- Prometheus / MLflow для мониторинга на VM
- A/B: epsilon-greedy vs exploit-only

## Риски

- Pipeline обновляется раз в 10 сек — при быстрых запросах грейдера модель может отставать
- `cleanup` должен полностью сбрасывать состояние (память, Redis, CSV)
- Один объект в новой сессии не считается релевантным — нужно исследовать интересы пользователя
- Бонус за скорость только при ≥30 баллов по основным метрикам
